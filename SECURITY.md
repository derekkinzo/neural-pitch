# Security Policy

## Reporting Security Issues

If you believe you have found a security vulnerability in NeuralPitch,
please report it **privately** by emailing
[derekkinzo@gmail.com](mailto:derekkinzo@gmail.com).

Please do **not** open a public GitHub issue, pull request, or discussion
thread for security vulnerabilities. Public reports give attackers a window
to exploit the issue before a fix is available.

When reporting, include as much of the following as you can:

- A description of the vulnerability and its impact.
- Steps to reproduce, ideally with a minimal proof of concept.
- The commit SHA or release in which you observed the issue.
- Any suggested mitigation, if you have one.

You will receive an acknowledgement of your report. Once the issue is
understood and fixed, the report and fix will be disclosed publicly along
with credit to the reporter, unless the reporter requests otherwise.

## Supported Versions

Only the latest commit on `main` is supported. There are no long-term-support
branches and no backports to older tags or releases. Security fixes land on
`main` and are picked up by the next build.
